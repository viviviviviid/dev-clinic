## 개념 설명
Go 언어의 가장 강력한 무기 중 하나는 '고루틴(Goroutine)'과 '채널(Channel)'을 통한 동시성 프로그래밍입니다. 순차적인 크롤러는 한 페이지를 처리한 후 다음 페이지로 넘어가기 때문에 대기 시간이 길어 비효율적입니다. 동시성 크롤러는 여러 개의 URL을 동시에 탐색하여 크롤링 속도를 획기적으로 높일 수 있습니다.

고루틴은 수 KB의 메모리만 차지하는 경량 스레드로, 수만 개를 동시에 실행해도 시스템에 큰 부하를 주지 않습니다. 하지만 무분별한 고루틴 생성은 타겟 서버에 과도한 트래픽을 유발(DoS)할 수 있으므로 '워커 풀(Worker Pool)' 패턴을 사용하여 동시 실행 개수를 제한하는 것이 중요합니다. 채널은 고루틴 간에 데이터를 안전하게 주고받는 통로 역할을 합니다. 이를 활용하면 메인 고루틴에서 크롤링할 URL을 분배하고, 작업 고루틴에서 결과를 수집하는 효율적인 파이프라인을 구축할 수 있습니다. 동시성 모델을 이해하는 것은 고성능 네트워크 애플리케이션을 설계하는 필수적인 과정입니다.

## 현재 과제
### 구현할 것 (HOLE)
1. **main.go - crawl(url string, wg *sync.WaitGroup, results chan<- string)**: 주어진 URL을 크롤링하고, 발견된 링크들을 채널을 통해 전달하는 함수를 작성합니다.
   - 왜 필요한가: 여러 페이지를 동시에 방문하여 데이터를 수집하기 위해 비동기 처리가 필요하기 때문입니다.
   - 단계별 접근: sync.WaitGroup을 통해 작업 완료를 추적 → 고루틴 내에서 crawl 호출 → 발견된 링크를 results 채널에 전달.
   - 사용할 것: goroutine, sync.WaitGroup, channel.

### 찾아서 고칠 것 (BUG)
1. **main.go - 동시성 제어 로직**: 현재 코드에서 `go crawl(url)`을 반복문 내에서 무제한으로 호출하고 있어, 수많은 URL 방문 시 'Too many open files' 오류나 타겟 서버의 차단(429 Too Many Requests)이 발생합니다.
   - 증상: 프로그램 실행 시 시스템 자원 과다 점유 혹은 서버로부터의 응답 거부.
   - 힌트: 버퍼가 있는 채널(Semaphore 역할)을 활용하여 동시에 실행되는 고루틴의 개수를 일정 수준(예: 5개)으로 제한하는 로직을 추가해야 합니다.

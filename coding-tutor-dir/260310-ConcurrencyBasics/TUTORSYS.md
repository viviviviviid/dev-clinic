# TUTORSYS

## 학습자 목표
Go 언어의 핵심인 고루틴(Goroutine)과 채널(Channel)을 이해하고, 이를 활용해 여러 작업을 동시에 안전하게 처리하는 동시성 제어 능력을 기릅니다. 복잡한 동기화 문제를 해결하며 Go만의 효율적인 동시성 모델을 실습합니다.

## 언어 & 환경
Go

## 학습 수준

## 개념 설명
Go 언어에서 동시성은 '공유하는 것보다 통신을 통해 메모리를 공유하라'는 철학을 바탕으로 합니다. 여기서 고루틴(Goroutine)은 Go 런타임이 관리하는 초경량 스레드입니다. 함수 앞에 `go` 키워드만 붙이면 메인 함수와 독립적으로 실행되는데, 수천 개의 고루틴도 적은 메모리로 운영할 수 있다는 점이 특징입니다.

문제는 여러 고루틴이 동시에 같은 변수에 접근할 때 발생하는 데이터 경합(Race Condition)입니다. 이를 해결하기 위해 채널(Channel)이 필요합니다. 채널은 고루틴 간의 파이프라인 역할을 하며, 데이터를 주고받으며 자연스럽게 실행 흐름을 동기화합니다. 예를 들어, 메인 고루틴이 작업 고루틴의 결과를 기다려야 할 때 채널을 통해 신호를 받으면, 복잡한 락(Lock) 없이도 안전하게 데이터를 주고받을 수 있습니다. 채널은 송신자와 수신자가 준비될 때까지 기다리는 '블로킹' 특성을 가지고 있어 동시성 제어에 매우 효과적입니다.

## 현재 과제
### 구현할 것 (HOLE)
1. **main.go - workerGroup**: 5개의 고루틴을 생성하여 각각 1초간 작업을 수행하고 완료 메시지를 출력하는 로직을 작성합니다.
   - 왜 필요한가: 여러 고루틴이 비동기로 실행될 때 메인 프로세스가 고루틴이 끝날 때까지 기다리지 않고 종료되는 문제를 해결하기 위해 필요합니다.
   - 단계별 접근: 1단계) sync.WaitGroup을 선언합니다. 2단계) 각 고루틴 시작 전 Add(1)을 호출합니다. 3단계) 고루틴 내부에서 작업 완료 후 Done()을 호출합니다. 4단계) 메인 함수에서 Wait()를 호출합니다.
   - 사용할 것: sync.WaitGroup, go 키워드, for 루프

### 찾아서 고칠 것 (BUG)
1. **main.go - goRoutineError**: for 루프 내부에서 `go func() { fmt.Println(i) }()` 형태로 호출할 때 발생하는 변수 캡처 문제를 해결하세요.
   - 증상: 출력 결과가 루프 변수의 최신값으로 모두 동일하게 출력되거나, 의도하지 않은 숫자가 반복되어 출력됩니다.
   - 힌트: 클로저(Closure)가 외부 변수를 참조할 때 발생하는 캡처 시점의 차이를 인지하고, 변수를 어떻게 인자로 전달할지 고민해보세요.
